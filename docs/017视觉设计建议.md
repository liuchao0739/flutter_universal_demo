## 1️⃣ 整体视觉风格定位

### 关键词

* 工程感
* 秩序感
* 架构感
* 克制、不花哨

### 禁用

* 渐变滥用
* 卡通插画
* 花哨动效

---

## 2️⃣ 页面版式规范（非常重要）

### 标准页面结构（80% 页面）

```
┌──────────────────────────┐
│  标题（36–40px，左对齐）  │
│  副标题（可选，16–18px）  │
│──────────────────────────│
│                          │
│  内容区（图 / 要点）     │
│                          │
│                          │
└──────────────────────────┘
```

**原则：**

* 所有标题左对齐
* 一页只讲一个观点
* 留白 ≥ 30%

---

## 3️⃣ 字体建议

### 中文

* 思源黑体 / 阿里巴巴普惠体

### 英文 & 数字

* Inter / Roboto Mono

### 字号参考

| 类型 | 字号    |
| -- | ----- |
| 标题 | 36–40 |
| 正文 | 18–22 |
| 注释 | 14–16 |

---

## 4️⃣ 配色方案（推荐）

### 主色

* 深蓝 / 深灰（#1F2937）

### 强调色

* 蓝色（#3B82F6）
* 橙色（仅用于关键指标）

### 背景

* 白 / 极浅灰

---

## 5️⃣ 架构图设计规范（重点）

### 不要画这种

❌ 复杂 UML
❌ 细节过多
❌ 代码级结构

### 推荐画法

```
┌──────── App ────────┐
│                     │
│   Core / Service    │
│   Feature           │
│                     │
└─────────────────────┘
```

* 方框
* 箭头
* 少字
* 强逻辑
