## 1️⃣ 项目总体架构说明

### 项目定位

> 本项目是一个 **Flutter 通用行业级母版工程**
> 目标：
>
> * 一次搭建，多行业复用
> * 支持 Android / iOS / 鸿蒙 Flutter
> * 内置完整 Demo，开箱即用
> * 支持 Mock → 生产平滑演进

---

## 2️⃣ 架构分层设计

```
┌────────────────────────────┐
│          App Shell         │
│  Theme / Language / Router │
└─────────────▲──────────────┘
              │
┌─────────────┴──────────────┐
│         Page Layer         │
│ Splash / Login / Home     │
│ Settings                  │
└─────────────▲──────────────┘
              │
┌─────────────┴──────────────┐
│         Demo Layer         │
│ Map / Scan / Share / etc  │
└─────────────▲──────────────┘
              │
┌─────────────┴──────────────┐
│       Service Layer        │
│ Api / Crash / Analytics   │
│ Storage / Map / Payment   │
└─────────────▲──────────────┘
              │
┌─────────────┴──────────────┐
│        Config Layer        │
│ Theme / Language / Env     │
│ Constants                 │
└────────────────────────────┘
```

---

## 3️⃣ 页面层职责（Page Layer）

| 页面           | 职责           |
| ------------ | ------------ |
| SplashPage   | 启动 / 引导      |
| LoginPage    | 登录 / 注册      |
| HomePage     | Demo 功能聚合    |
| SettingsPage | 配置 / 缓存 / 权限 |

**原则**：

* 页面只处理 UI + 路由
* 不直接操作 SDK
* 所有逻辑交给 Service

---

## 4️⃣ Demo 层职责（Demo Layer）

Demo 层用于：

* 展示功能效果
* 验证 SDK 集成
* 提供可交互示例

| Demo              | 对应 Service        |
| ----------------- | ----------------- |
| MapDemo           | MapService        |
| ScanDemo          | ScanService       |
| ShareDemo         | ShareService      |
| CarouselDemo      | ApiService        |
| ThemeLanguageDemo | Theme / Language  |
| PermissionDemo    | PermissionService |

---

## 5️⃣ Service 层职责（核心）

**Service 是整个项目的“能力中心”**

原则：

* 一个 Service = 一类能力
* 不依赖 UI
* 可被 Demo / Page / 后台任务复用

---

## 6️⃣ Config 层职责

Config 层是 **全局共享、只读或轻变配置**：

* Theme：主题
* Language：多语言
* Env：环境
* Constants：Key / 常量
